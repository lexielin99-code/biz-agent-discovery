# Capability Library — biz-agent-discovery

内部参考，不直接展示给用户。根据用户在阶段一的确认情况，匹配最相关的 1-2 个场景用于 demo 生成。

---

## 营销方向

| 场景 | 主动索取话术 | Demo 形式 | Remix 方向 |
|------|-------------|-----------|-----------|
| 月度大盘复盘报告 | 「把你的数据表格发给我，我来生成一个可视化复盘」 | 交互式 HTML 趋势图，含渠道对比和 MoM 变化 | 接不同渠道数据源、自定义 KPI 维度 |
| 历史方案策略沉淀 | 「把你过去的方案/brief 发给我，我来梳理你的策略逻辑和 wording 风格」 | 策略摘要文档 + 高频词/句式提炼 | 可扩展为个人素材库、品牌语调规范 |
| 内容批量生产 SOP | 「告诉我你的内容生产流程，或者把你用过的模板发给我，我来做成可视化 SOP」 | 流程图 + 分平台模板骨架 | 不同平台/话题的变体模板，可复用结构 |
| 社媒内容批量扩展 | 「给我一个核心 idea 或者一段素材，我帮你扩展成多平台版本」 | 一稿 → 微博/小红书/公众号/LinkedIn 适配输出 | 不同平台语气、发布节奏、话题角度 |
| 竞品内容监测简报 | 「把你这周看到的竞品内容（截图/链接/文字）发给我，我来整理成结构化简报」 | 结构化竞品简报，含主题提炼/渠道/互动观察/洞察摘要 | 监测频率、品牌数量、分析维度可调。**⚠️ 数据来源说明：Claude 默认不主动抓取网络数据，需用户提供素材（截图/链接/摘录）；如已启用 web-access skill，可指定 URL 让 Claude 读取内容，但不能自动持续监测** |
| 邮件/私信 A/B 文案 | 「给我你的受众描述和核心诉求，我来出多个版本让你比较」 | 3 版文案对比输出，含差异化分析 | 不同受众群体、不同 CTA、不同情绪基调 |
| 用户反馈/评论挖掘 | 「把评论/问卷/NPS 数据给我，我来提炼情绪分布和关键主题」 | 情绪分类 + 高频词提炼 + 问题优先级排序 | 接平台评论/NPS/用研录音/客服记录 |
| 投放复盘报告 | 「把广告后台数据导出给我，我来整理成渠道对比分析」 | HTML 渠道对比 + 归因骨架 + 异常标注 | 自定义归因模型、维度拆分（地区/人群/素材） |
| 媒体/KOL 沟通草稿 | 「给我项目 brief 和你要触达的对象描述，我来批量生成个性化 pitch 草稿」 | 多版本 pitch 模板 + 定制化变量说明 | 接 KOL 画像数据批量生成、不同合作形式 |

## 电商运营方向

| 场景 | 主动索取话术 | Demo 形式 | Remix 方向 |
|------|-------------|-----------|-----------|
| 销售数据大盘看板 | 「把平台导出的销售表给我，我来做一个可交互的看板」 | HTML 看板，含趋势图 + 异常自动标注 + Top/Flop 列表 | 接多平台数据、自定义维度、下钻到 SKU |
| 运营规范/工作流沉淀 | 「把你的工作流程描述给我，或者发给我你积累的文档，我来整理成规范」 | 结构化 SOP 文档，含步骤说明、注意事项、决策节点 | 可进一步生成新人培训材料、执行 checklist |
| 库存健康度报告 | 「把库存数据表给我，我来生成一个带风险预警的可视化报告」 | 可视化风险预警表，含安全库存线和滞销标注 | 不同品类分组、自定义预警阈值、多仓合并 |
| 大促节奏规划 | 「把历史大促数据和今年目标给我，我来帮你做节奏规划和执行清单」 | 日历视图 + 分阶段执行清单 + 资源分配建议 | 接不同大促周期、不同平台节奏、多品类协调 |
| 商品文案批量优化 | 「把 SKU 列表和改写方向给我，我来批量出改写版本」 | 改写前后对比示例 + 改写逻辑说明 | 批量处理多 SKU、统一品牌语调、分品类定制 |
| 客服 FAQ 生成 | 「把历史工单或评论数据给我，我来生成 FAQ 文档骨架」 | 问答文档骨架，按问题类型分类 | 接不同类目、不同渠道来源、定期更新机制 |
| SKU/品类表现分析 | 「给我数据，我来拆分成地区/渠道/时间段的对比分析」 | 多维对比报告，含同比/环比和归因分析 | 细分到 SKU 级别、自定义对比维度 |
| 供应商沟通邮件 | 「把采购需求或库存问题描述给我，我来起草专业邮件」 | 邮件草稿模板，含正式用语和关键信息结构 | 接批量采购数据生成、不同合作场景（催货/投诉/谈判） |

## 通用（营销和电商都适用）

| 场景 | 主动索取话术 | Demo 形式 | Remix 方向 |
|------|-------------|-----------|-----------|
| 汇报材料 → HTML 交互报告 | 「把你的数据和汇报要点给我，我来生成一个比 PPT 更好看的交互报告」 | 替代 PPT 的 HTML 报告，含可交互图表和数据 | 不同汇报对象风格、数据嵌入深度、品牌色适配 |
| 工作流/业务规范沉淀 | 「把你的工作方式描述给我，或者发过去的经验文档，我来帮你梳理成规范」 | 结构化工作流文档，含决策逻辑和例外处理 | 可进一步生成培训材料或团队共享规范 |
| 会议纪要 → 行动项 | 「把会议录音/记录发给我，我来整理结构化行动项和 owner」 | 带 owner 和 deadline 的行动清单，含背景摘要 | 不同会议类型（周会/复盘/对齐）、跟进提醒机制 |
| 数据 → 管理层叙事 | 「把数据给我，告诉我汇报对象是谁，我来帮你把数字转成有说服力的故事」 | Narrative 文字 + 配套图表，重点突出结论 | 不同汇报对象（老板/跨部门/投资方）风格 |
| 竞品研究摘要 | 「告诉我你关注的竞品和你最想了解的维度，我来聚合成一份简报」 | 多维度竞品简报，含结构化对比和洞察提炼 | 不同关注维度（产品/营销/定价/用户口碑） |

## 能力库使用说明

**Demo 选取原则：**
1. 优先匹配用户在阶段一确认了「是」的场景
2. 如果用户有自由发挥（如提到具体痛点），优先响应他们提到的内容
3. 选 1-2 个场景就够，不要一次展示太多

**主动索取的时机：**
- 确认用户的方向后立刻提出，不要等用户问「那我怎么开始」
- 语气自然，像在跟同事说话，不是在给流程

**Demo 质量标准：**
- 用虚构但合理的样本数据，让 demo 看起来像真实场景
- HTML 类 demo 要可以直接在浏览器打开
- 每个 demo 后必须跟随开放性说明
